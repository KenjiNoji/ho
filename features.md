# Pomodoro Timer App Features

## 目的

このドキュメントは、ポモドーロタイマー Web アプリケーションを実装するにあたって必要となる機能を整理したものである。

実装優先度を明確にするため、機能は以下の 4 区分で整理する。

- 初版で必須の機能
- 初版でほぼ必須に近い補助機能
- 将来追加しやすい拡張機能
- 技術的に明示して実装すべき基盤機能

## 初版で必須の機能

### 1. タイマー表示機能

- 現在の残り時間を分と秒で表示する
- 現在のモードに応じて表示内容を切り替える
- タイマー終了時に表示が正しく 00:00 になる

### 2. タイマー制御機能

- タイマーを開始できる
- タイマーを一時停止できる
- 一時停止後に再開できる
- タイマーをリセットできる

### 3. モード切替機能

- 作業モードに切り替えられる
- 短い休憩モードに切り替えられる
- 長い休憩モードに切り替えられる
- モード切替時に表示時間が対応する設定値へ更新される

### 4. ポモドーロサイクル管理機能

- 作業セッションの完了回数を管理する
- 規定回数ごとに長い休憩へ遷移する
- それ以外は短い休憩へ遷移する

### 5. 設定変更機能

- 作業時間を変更できる
- 短い休憩時間を変更できる
- 長い休憩時間を変更できる
- 長い休憩へ入る間隔を変更できる

### 6. 状態保持機能

- 設定値を保存できる
- ページ再読み込み後に設定を復元できる
- 必要に応じて現在モードや残り時間を復元できる

### 7. 入力バリデーション機能

- 不正な時間設定を拒否できる
- 範囲外の long break 間隔を拒否できる
- 入力エラー時にユーザーへ分かる形で通知できる

### 8. 画面更新機能

- タイマー状態に応じて表示を更新する
- 現在モードを UI に反映する
- ボタンの状態を適切に切り替える
- 完了回数や関連表示を更新する

### 9. 基本レスポンシブ対応

- デスクトップでレイアウトが崩れない
- モバイルで主要操作がしやすい
- 画面サイズに応じてタイマー表示と操作部が適切に配置される

### 10. Flask の画面配信機能

- トップページでアプリ画面を返す
- HTML テンプレートを配信する
- CSS と JavaScript の静的ファイルを配信する

## 初版でほぼ必須に近い補助機能

### 1. タイマー終了通知機能

- 作業終了時に通知する
- 休憩終了時に通知する
- 音または画面表示、可能ならブラウザ通知で知らせる

### 2. 自動開始設定機能

- 休憩を自動開始するか設定できる
- 次の作業を自動開始するか設定できる

### 3. 現在モードの視覚的区別

- 作業中、短い休憩中、長い休憩中でラベルを変える
- 必要に応じて配色やアクセントも切り替える

### 4. 完了回数表示機能

- 現在の完了回数を表示する
- long break まで残り何回かが分かる構成にできる

## 将来追加しやすい拡張機能

### 1. 統計表示機能

- 当日の完了回数を表示する
- 総作業時間を表示する
- 日別の利用傾向を表示する

### 2. セッション履歴保存機能

- セッション完了時刻を記録する
- 作業履歴を一覧表示できる

### 3. サーバー保存機能

- 設定をサーバー側へ保存する
- 履歴や統計をサーバー側で管理する

### 4. タスク連携機能

- 現在取り組むタスク名を入力できる
- セッションとタスクを紐付けられる

### 5. テーマ切替機能

- 複数の UI テーマを切り替えられる
- ライトテーマやダークテーマを選べる

### 6. 通知詳細設定機能

- 通知の有効無効を切り替えられる
- 音量や通知方法を調整できる

## 技術的に明示して実装すべき基盤機能

### 1. タイマーエンジン

- 残り時間計算を担う
- セッション終了判定を担う
- モード遷移を担う
- long break 判定を担う

### 2. Clock 抽象

- 現在時刻の取得を差し替え可能にする
- テスト時に固定時刻や疑似経過時間を使えるようにする

### 3. Storage 抽象

- localStorage を直接参照しない構成にする
- 将来のサーバー保存実装へ差し替えやすくする

### 4. Renderer

- state を受け取って UI を描画する
- DOM 更新とタイマーロジックを分離する

### 5. Flask app factory

- create_app(config=None) 形式でアプリを生成する
- テスト設定を差し込めるようにする

### 6. Route と Service の分離

- Flask route には入出力処理のみを書き、実処理は service へ委譲する
- API の単体テストと将来拡張をしやすくする

### 7. 設定バリデーション関数

- 時間設定や間隔設定の妥当性を独立関数で検証する
- 境界値テストしやすい構成にする

## 優先順位付き実装順

1. Flask による画面配信
2. タイマー表示
3. 開始、一時停止、再開、リセット
4. モード切替とサイクル管理
5. 設定変更と入力バリデーション
6. localStorage による保存と復元
7. 通知機能
8. 完了回数表示
9. 統計表示とサーバー API

## MVP の定義

最小実装として成立させるために必要なのは以下である。

- タイマー表示
- 開始、一時停止、再開、リセット
- 作業、短い休憩、長い休憩の切替
- ポモドーロサイクル管理
- 設定変更と入力バリデーション
- 設定または状態の保存と復元
- 基本的な通知
- レスポンシブな UI
- Flask による画面配信

## 補足

初版では、統計、履歴、SQLite、複数端末同期までは必須にしない。
まずはブラウザ中心で完結する構成とし、必要になった段階で Flask API とサーバー保存機能を拡張する方針とする。
